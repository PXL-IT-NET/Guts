Write monitored tests with JUnit 5

The JUnit 5 client uses a TestExecutionListener to capture test execution and submit results to the GUTS REST API.

Installation

Maven

Add this test dependency to pom.xml:

<dependency>
    <groupId>be.pxl.guts</groupId>
    <artifactId>client-junit-5</artifactId>
    <version>1.0-SNAPSHOT</version>
    <scope>test</scope>
</dependency>
Gradle

Add this dependency to build.gradle:

dependencies {
    testImplementation 'be.pxl.guts:client-junit-5:1.0-SNAPSHOT'
      }

Configuration (guts.json)

On first test run, a guts.json file is created in the project root folder.

  • baseUrl: URL of the GUTS REST API (e.g. http://guts-api.pxl.be/).
  • webUrl: URL of the GUTS web application (e.g. http://guts-web.pxl.be/).
  • sourceDirectory: source path from the root folder.
  • testDirectory: test path from the root folder.

Default file content:

{
  "baseUrl": "http://localhost:5000/",
  "webUrl": "localhost:4200/",
  "sourceDirectory": "src/main/java/",
  "testDirectory": "src/test/java/"
}

Build To Local Maven

Run gradle publishToMavenLocal in the guts-client-junit-5 folder to publish the client locally.

Class-level annotation

@GutsFixture(options)

Put this annotation on the test class to identify where the test results belong in GUTS.

@GutsFixture(courseCode, chapterCode, exerciseCode, sourceCodeRelativeFilePaths)

@GutsFixture options

  • courseCode: unique identifier of the course.
  • chapterCode: identifier of the chapter.
  • exerciseCode: identifier of the exercise.
  • sourceCodeRelativeFilePaths: relative Java source file path (or semicolon-separated paths) to send source code along with test results.

Test-level conventions

  • Supported test annotations include @Test, @RepeatedTest, and @ParameterizedTest.
  • Supported parameterized sources include @EnumSource, @ArgumentsSource, @MethodSource, @ValueSource, @CsvSource, and @CsvFileSource.
  • Use @DisplayName and assertion messages to provide clear, student-friendly feedback in GUTS.
