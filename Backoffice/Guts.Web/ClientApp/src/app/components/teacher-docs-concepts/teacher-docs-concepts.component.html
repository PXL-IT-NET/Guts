Monitor Your Students' progress with GUTS

GUTS helps teachers turn automated tests into actionable feedback for students and teams, while keeping result collection centralized and secure.
Also, GUTS offers a peer assessment tool to evaluate each other when working in teams.

What you gain by using GUTS monitoring

You can create automated tests without GUTS, but the platform adds clear value for teaching and coaching. GUTS currently supports C# and Java, and additional languages can be added over time.

Centralized progress tracking

Every monitored testrun sends results to the backoffice so both students and teachers can track progress over time.

Motivation through comparison

Students can compare results with average results of other students to identify what to improve next.

Anti-tampering protection

Hash validation helps ensure students cannot modify teacher tests and still submit accepted results.

Exercise work vs project work

Individual exercises

In GUTS courses contain chapters, and each chapter contains one or more exercises.

When a teacher runs the tests, missing chapters and exercises are automatically created in the backoffice based on metadata in your test attributes.
Test results are accumulated per student per exercise.

Team projects

Courses can also contain projects that are split into components.

When a teacher runs the tests, missing projects and components are automatically created in the backoffice based on metadata in your test attributes.
Test results are accumulated per team per component.
