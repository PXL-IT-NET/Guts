Concepts

1. Automated feedback

One of the key concepts in GUTS is automated feedback, which allows students to receive immediate responses to their code of exercises or projects.

As a teacher, you can create coding exercises and projects that are accompanied by automated tests. When students run these tests, they receive instant feedback on whether their code is correct or if there are any issues that need to be addressed. This gives students insights into how to improve their code and better understand the concepts being taught.

The automated tests are written by the teacher, using GUTS extensions of popular testing frameworks such as xUnit or NUnit.
GUTS currently supports C# and Java, but more languages could be added in the future.

Although a teacher can always create automated tests in an exercise or project without the GUTS tools, using the GUTS system offers several advantages:

  • Centralization of test results: on each test run, the results are sent to the GUTS server. Students can see their test results in the GUTS dashboard, and teachers can monitor the progress of their students and identify common issues or areas where students may be struggling.
  • Gamifications: compare your test results with the average results of other students.
  • Protection against cheating: the GUTS system ensures that the tests are not altered by students.

1.1 Coding exercises

In GUTS there are 2 kinds of coding exercises: individual exercises and projects:

1.1.1 Individual exercises

A course can contain multiple chapters which contain exercises

As a teacher, you can add chapters and exercises to a course by running the automated tests of a coding exercise you created. When the test results are sent to the GUTS server, the system will automatically create a chapter and exercise (if they don't exist yet) based on the metadata you provided in the test code.

The test results are then accumulated per student per exercise.

1.1.2 Projects

Next to chapters, a course can also contain multiple projects. A project is made out of multiple components. Each component represents a part of the project (typically a certain class).

As a teacher, you can add projects and its components to a course by running the automated tests of the coding project you created. When the test results are sent to the GUTS server, the system will automatically create a project and component (if they don't exist yet) based on the metadata you provided in the test code.

Students work in teams (registered in GUTS) on the projects, and they run the tests of the project to get feedback on their code. The test results are then accumulated per team per component.
